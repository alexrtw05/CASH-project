// WHAT IS A PER-LETTER CAESAR CIPHER

A Caesar cipher shifts each letter forward in the alphabet by some amount. A per-letter Caesar uses a different shift for every position - so trying all 26 shifts on the whole word will not recover the plaintext. You need the formula and the actual numbers.

ORIGINAL WORD
↓ shift_i = (shift_base + i*7) mod 26 ↓
ENCODED (each letter pushed)
↓ subtract the same shifts to decode ↓
DECODED
The base shift is built from three numbers: shift_base = (pixel_x + pixel_y + venus.phase + decoy_sum) mod 26
You need the image (cv2) AND ephem to reconstruct it. Neither alone is enough.
// FINDING THE TEAL PIXEL AND THE GHOSTS

OpenCV stores images in BGR order (Blue, Green, Red) — not RGB. The real cyan pixel has B=255, G=255, R=0 exactly. Decoy pixels look similar but have R > 0 (specifically R in the range 1-30). You need both: the cyan pixel's coordinates, AND the sum of the decoy red values.

# Find the exact cyan pixel
real_mask = (img[:,:,0] == 255) & (img[:,:,1] == 255) & (img[:,:,2] == 0)
ys, xs = np.where(real_mask)
pixel_x = int(np.median(xs))
pixel_y = int(np.median(ys))

# Sum the red values of the decoy near-cyan pixels
decoy_mask = (img[:,:,0] == 255) & (img[:,:,1] == 255) & (img[:,:,2] > 0) & (img[:,:,2] < 50)
decoy_sum = int(img[decoy_mask, 2].sum())
  • Load the image with cv2.imdecode from the base64 string
  • Real cyan pixel: B==255 AND G==255 AND R==0 (decoys have R>0)
  • Use np.where to get coordinates, then take the median (cross shape = multiple pixels)
  • Decoys: B==255 AND G==255 AND 0 < R < 50; sum their red channels for decoy_sum
  • Both numbers go into your shift formula
