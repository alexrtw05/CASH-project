// WHAT IS A TRANSPOSITION CIPHER

A transposition cipher does not change the letters - it rearranges them. The same letters, different order. To decode it, you need to know exactly how they were rearranged.

ORIGINAL WORD
↓ permutation applied ↓
ENCODED (SCRAMBLED)
↓ reversed to decode ↓
DECODED
The permutation comes from a key: key = pixel_x * pixel_y + int(venus.phase)
You need the image (cv2) AND ephem to reconstruct it. Neither alone is enough.
// FINDING THE TEAL PIXEL

OpenCV stores images in BGR order (Blue, Green, Red) — not RGB. The cyan pixel has B=255, G=255, R=0 exactly. Decoy pixels look similar but have R > 0.

# Find the exact cyan pixel
mask = (img[:,:,0] == 255) & (img[:,:,1] == 255) & (img[:,:,2] == 0)
ys, xs = np.where(mask)
pixel_x = int(np.median(xs))
pixel_y = int(np.median(ys))
  • Load the image with cv2.imdecode from the base64 string
  • Create a boolean mask: pixels where B==255 AND G==255 AND R==0
  • Use np.where to get the coordinates, then take the median (cross shape = multiple pixels)
  • These x, y values go into your key formula
