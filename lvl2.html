// CV2 STORES IMAGES AS BGR - NOT RGB

OpenCV uses BGR channel order — Blue first, then Green, then Red. This catches almost everyone. When you do img[y, x], you get [B, G, R].

BLUE
img[y,x,0]
B
This is your planet distance value.
Read this channel.
GREEN
img[y,x,1]
G
Always 255 for planet markers.
Use to find them.
RED
img[y,x,2]
R
Always 255 for planet markers.
Use to find them.
Planet markers: G==255 AND R==255 AND B != 255
Pure white decoys: B==255 AND G==255 AND R==255
So filter: (img[:,:,1]==255) & (img[:,:,2]==255) & (img[:,:,0]!=255)
// FINDING CLUSTER CENTERS

Each planet marker is a 3x3 patch of pixels, not a single pixel. You need to group the matching pixels into 4 clusters and find each cluster's center.

from scipy import ndimage

# Find all planet marker pixels
mask = (img[:,:,1]==255) & (img[:,:,2]==255) & (img[:,:,0]!=255)

# Label connected groups of pixels
labeled, n = ndimage.label(mask)

# Get center and blue channel of each cluster
clusters = []
for i in range(1, n+1):
  ys, xs = np.where(labeled == i)
  cy, cx = int(np.median(ys)), int(np.median(xs))
  blue = int(img[cy, cx, 0])
  clusters.append((cx, cy, blue))
  • Filter pixels with the mask above — this gives you all marker pixels
  • Use scipy.ndimage.label to group connected pixels into clusters
  • For each cluster, take the median x and y as the center
  • Read the blue channel at that center pixel
  • Match each cluster to a planet using the position formula
